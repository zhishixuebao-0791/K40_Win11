## Interpretation

This is currently an ACPI runtime enumeration problem, not a driver-injection or WDAC problem.

Phase10 proved that at least one remapped subsystem root can reach Windows: `SPSS` became `QCOM068D` and selected the Surface 8280 `qcsubsys` package. However, the other three intended roots did not appear in PnP at all, so driver injection cannot help them yet.

The Phase10 build script validates that strings exist in compiled AML, but that is not enough. Phase11 must validate and patch the actual active `Device(SCSS)`, `Device(ADSP)`, and `Device(CDSP)` runtime paths, including `_STA`, `_DEP`, and parent dependency behavior.
